软工大作业
课程项目内容：
本项目拟定开发一个具有展示功能的、类似朋友圈的多媒体展示小软件。该软件包含一个用户使用的小程序前端和一个管理员使用的Web后台。
1.	小程序前端
用户可以上传图片、视频、文本，以及查看、修改、删除自己上传的内容。
上传的内容可以增加标签，便于通过日期、标签等多种形式进行检索
其他人可以对内容进行评价（对内容打分）、评论（类似朋友圈评论）
用户可以设置自己的登录密码
加分项：实现好友功能、集成敏感词过滤、视频流处理、对内容集成大模型处理等功能。
2.	Web后台
      从管理员的角度，对所有用户上传的内容进行管理。
      对使用系统的用户进行管理，包括查询、删除等
      对用户提交的内容进行管理，包括查询、删除等
      加分项：提供各种用户使用、活跃情况的统计分析图。

要求：整体界面良好，格式工整，代码风格良好。

用户故事：
用户故事一：内容分享与互动
角色：普通用户李明
目标：记录和分享校园生活瞬间，与朋友互动
场景：
李明参加校园音乐节后，想要分享精彩时刻
他打开小程序，点击"+"按钮上传拍摄的1段视频和2张照片
为内容添加标签：#校园音乐节
编写描述文字："今天的音乐节太棒了！"
*发布后，好友们可以看到内容并互动：
*王小红打了4星评分并评论："第二个乐队确实厉害！"
*张三回复王小红的评论："同意！"
*李明收到通知，可以实时看到朋友的评分和评论
验收标准：
用户可以成功上传多媒体内容
标签系统正常工作，便于后续检索
*好友可以正常进行评分和评论互动

用户故事二：后台内容管理与数据分析
角色：系统管理员张老师
目标：维护平台内容安全，分析用户行为数据
场景：
张老师登录Web管理后台，首页显示系统概览：
待审核内容：12条
*系统警告：3条敏感内容待处理
首先进入"内容审核"页面：
发现一条包含不当言论的评论，点击"删除"并选择"违规言论"原因
批量审核通过5条符合规范的内容
切换到"数据统计"页面：
*查看"用户活跃度分析"，发现周末使用量明显上升
*通过"热门标签分析"发现"#毕业季"标签使用频率最高
*导出上周的内容上传报表用于工作汇报
在"用户管理"中：
处理一个发布广告的账号，进行7天禁言处罚
查看用户反馈，发现视频上传速度的优化需求
验收标准：
管理员能够有效审核和管理用户内容
*数据统计图表准确反映系统运行状况
管理操作记录完整可追溯
*能够快速识别和处理违规内容
